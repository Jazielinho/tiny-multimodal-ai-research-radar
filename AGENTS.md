# Agent Instructions

This repository is part of the Tiny Multimodal AI Research Radar sprint.

## Mission

Help build a small, professional and explainable AI research assistant. Prioritize clarity, traceability and evaluation over shiny features.

## Guardrails

- Do not expand scope without documenting the decision.
- Prefer small working increments over large rewrites.
- Every feature should have a minimal test, demo or example.
- RAG answers must include traceability to source chunks.
- Multimodal outputs must document uncertainty and limitations.
- Avoid hidden magic. Make decisions explicit in `docs/decisions/`.
- Do not commit private papers, credentials, API keys, local model weights or personal data.

## Preferred workflow

1. Create or update an issue.
2. Make a small change.
3. Add or update docs.
4. Run tests.
5. Write a short weekly note.
6. Prepare a clean PR.

## Sprint principle

This is not a race to build everything. It is a disciplined sprint to build evidence of applied AI engineering skill.
